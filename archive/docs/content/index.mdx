# State Designer

State Designer is a state management library for JavaScript and TypeScript.

As the name suggests, the library treats state as a _design problem_. If you're not exactly sure how the state of a user interface should work, then State Designer can help you figure it out.

- [Watch a Video](https://www.youtube.com/watch?v=HfZCcg9_3t8) and see the library in action
- [Explore examples](https://op1qk.csb.app/) in React and TypeScript.
- [Learn more](/designing-state) about what it means to design for state.

```jsx live=true
function Counter() {
  const state = useStateDesigner({
    data: { count: 0 },
    initial: 'stopped',
    states: {
      stopped: {
        on: { TOGGLED: { to: 'running' } },
      },
      running: {
        on: { TOGGLED: { to: 'stopped' } },
        repeat: {
          onRepeat: 'incrementCount',
          delay: 0.2,
        },
      },
    },
    actions: {
      incrementCount(data) {
        data.count++
      },
    },
  })

  return (
    <Grid columns="auto 1fr">
      <Button onClick={() => state.send('TOGGLED')}>
        {state.whenIn({
          running: 'Stop',
          stopped: 'Start',
        })}
      </Button>
      <Heading as="h1">{state.data.count}</Heading>
    </Grid>
  )
}
```

## Inspiration

State Designer is heavily inspired by [xstate](https://github.com/davidkpiano/xstate). Note that, unlike xstate, State Designer does not adhere to the [scxml spec](https://en.wikipedia.org/wiki/SCXML).
